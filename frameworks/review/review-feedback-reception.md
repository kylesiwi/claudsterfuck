# Review Feedback Reception

- Verify feedback before implementing it.
- Restate unclear requirements or ask questions before touching code.
- Do not perform agreement for social reasons.
- Push back when feedback is technically wrong for this codebase.
- Check whether a suggested "proper" feature is actually needed before adding it.
- Implement review items one at a time and verify each one.

If feedback is correct:

- fix it
- state the change plainly

If feedback is wrong:

- explain why with codebase-specific reasoning
