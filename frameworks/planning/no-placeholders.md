# No Placeholders

- Do not write TODO, TBD, "implement later," or "add appropriate validation."
- If a step changes code, provide the actual code shape or concrete implementation guidance.
- If a step needs a command, give the exact command.
- If a step needs a test, describe the exact behavior to verify.
- Repeat critical details instead of relying on "same as above" when that would create ambiguity for the implementer.
